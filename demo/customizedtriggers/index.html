Explanations

There are 2 features in this tour.

First feature: Set 'dragAndDrop' attribute to true on the step where DnD is enabled. On drag event, the framework will release hold and wait till the target is dropped at which point the tour will be resume. Recommendation: on the step after DnD, you should set 'prerequisites' to check if user drop the target at correct location (return false when not). This feature allows framework to decide whether next step is valid or it should stay on the current step until user drops correctly.

Second feature: As you may have seen, the flow can be triggered by a button in your application. To do this, you just need to give 'nextStepTrigger' attribute the JQuery selector string. You can also provide a shortcut '@target@' which means that the trigger is the target of the step so that you don't have to rewrite the whole selector string.
