Explanations

There are 2 methods to deal with modal in this tour.

First feature: Set 'modal: true' attribute and the framework will make sure that its components will be up-to-date to your modal's position. However, this method takes a bit of processing time (debounce every 0.2 seconds).

Second feature: Along with 'modal: true', you can also set 'scrollLock: true' attribute to make scrolling disabled for that step. This will keep the modal in place which improve performance a little bit. However, do not use this attribute if you need users to scroll.



NOTICE: The overlay cannot cover the modal (usually 'fixed' position) as it can cover static components in the page. Due to this limitation, the problems might occur when users click on unwanted buttons in the modal and break the flow.


HINTS: You can create a 'prerequisites' function that would disable unwanted buttons in your application at the beginning of the step, then restore them after. Or you can just hope that users will follow your step description.
