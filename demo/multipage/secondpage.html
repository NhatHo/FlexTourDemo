Explanations

There are 2 features in this tour.

Multipage feature: On the step which would trigger a new page, you should set 'multipage: true' attribute. When user triggers the transition, the framework uses LocalStorage to keep the current step number and tour id. When the new page is loaded, the application should provide FlexTourJS with the same tour objects and the app must trigger 'run' action. The framework will automatically pick up the next step and run as normal.

Save Point feature: When the 'saveOnExit: true' attribute is set, whenever the tour is ended abruptly (pressing ESC or clicking on X button or clicking on overlay) the framework will save the current step number and tour id. So that when user re-run the same tour, the framework will pick up where it was left off. However, as showed in the demo, some steps are not showed when the page started (modal, dropdown menus, etc.). 'savePoint: true' attribute should be set on the step where you can be sure that it will exist so that the tour will resume on that step.
